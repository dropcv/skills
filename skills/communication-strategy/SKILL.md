# 候选人沟通策略

要联系一个候选人时，准备怎么开场、聊什么、注意什么。

## 何时用

- 准备第一次触达某候选人（电话、微信、邮件）
- 沟通卡住了，想换个角度推进
- 要给团队成员一份「这个候选人怎么聊」的脚本

## 前置：接入 DropCV 数据能力

本技能负责沟通方法，候选人数据靠 DropCV 数据工具拿。先接入 DropCV MCP 或
Skills API（见 https://docs.dropcv.work/integration/mcp ）。会用到：

- `candidate_profile_read` — 按 candidate_id 读候选人完整档案
- `candidate_search` — 不知道 ID 时按关键词/姓名先搜

## 工作流

1. 读候选人档案（`candidate_profile_read`）。
2. 明确当前沟通阶段（首次触达 / 跟进 / 临门一脚）和已知背景。
3. 按下面方法论给出沟通策略。

## 方法论

候选人不是在等你 —— 沟通的第一目标是「让他愿意接着聊」。

- **开场话术（2-3 条）**：用候选人档案里的具体信息做钩子（他的项目、他的公司、他可能的痛点），让对方觉得你是认真看过他、而不是群发。给 2-3 个不同角度的版本。
- **追问方向（2-3 条）**：第一轮该摸清什么 —— 看机会的真实意愿、对下家的核心诉求（钱、平台、稳定、成长）、决策链（谁拍板、有没有在看别的）。
- **风险提示（1-2 条）**：这个候选人沟通里容易踩的雷 —— 比如薪资预期可能过高、可能只是随便看看、可能有 counter-offer 风险 —— 提前提醒。

语气：真诚、专业，站在候选人职业发展的角度，不画饼、不施压。

## 输出格式

```
**沟通目标**：<这一轮要达成什么>

**开场话术**
1. ...
2. ...

**追问方向**
- ...

**风险提示**
- ...
```
