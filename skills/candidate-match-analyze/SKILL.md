# 候选人匹配分析

评估单个候选人对单个 JD 的匹配程度，给出结构化判断。

## 何时用

- 拿到一个候选人 + 一个 JD，想知道「合不合适、哪里合适、哪里有风险」
- 推荐前先自己过一遍匹配度
- 决定要不要约这个候选人沟通某个岗位

## 前置：接入 DropCV 数据能力

本技能负责匹配判断方法，候选人数据靠 DropCV 数据工具拿。先接入 DropCV MCP 或
Skills API（见 https://docs.dropcv.work/integration/mcp ）。会用到：

- `candidate_profile_read` — 按 candidate_id 读候选人完整档案
- `candidate_search` — 不知道 ID 时按关键词/姓名先搜
- `jd_parse` — 解析目标 JD 文本

## 工作流

1. 读候选人档案（`candidate_profile_read`）。
2. 解析目标 JD（`jd_parse`）成结构化要求。
3. 逐条把 JD 要求和候选人背景对照，按下面方法论输出。

## 方法论

匹配分析要回答：**哪里对得上、哪里对不上、还有什么没搞清楚。**

- **核心匹配点**：JD 的硬要求里，候选人明确满足的。每条说清「JD 要什么 + 候选人有什么」。
- **匹配风险**：JD 要求里候选人不满足、或不确定满足的 —— 技能缺口、经验深度不够、行业不对口、职级跨度大等。不要回避。
- **待核实问题**：档案信息不足以判断的点，列成具体问题，留给后续沟通/面试去问。
- 给一个**匹配结论**：一句话说清推进价值（强烈推荐推进 / 可推进但需补充信息 / 匹配度低不建议）。

## 输出格式

```
**匹配结论**：<一句话>

**核心匹配点**
- <JD 要求 → 候选人对应>
...

**匹配风险**
- ...

**待核实**
- ...
```
