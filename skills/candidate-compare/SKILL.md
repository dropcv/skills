# 候选人对比

把多个候选人放在一起横向对比，帮用户（或客户）做取舍。

## 何时用

- 手上有 2 个及以上候选人竞争同一个岗位
- 客户问「这几个里你最推荐谁」
- 要给客户一份候选人对比表

## 前置：接入 DropCV 数据能力

本技能负责对比方法，候选人数据靠 DropCV 数据工具拿。先接入 DropCV MCP 或
Skills API（见 https://docs.dropcv.work/integration/mcp ）。会用到：

- `candidate_profile_read` — 按 candidate_id 读每个候选人完整档案
- `candidate_search` — ID 不全时按关键词/姓名补
- `jd_parse` — 解析目标 JD 文本（可选）

## 工作流

1. 拿到要对比的候选人 ID 列表（不全的话用 `candidate_search` 补）。
2. 逐个 `candidate_profile_read` 读档案。
3. 有目标 JD 就 `jd_parse`，对比围绕岗位展开；没有就做通用对比。
4. 按下面方法论对比。

## 方法论

对比的价值在于**突出差异**，不是各夸一遍。

- 先定**对比维度**：围绕 JD 的核心要求（硬技能、经验深度、行业匹配、稳定性、薪资预期、到岗速度等）选 4-6 个真正会影响决策的维度。
- 每个候选人给：**相对强项**（比别人强在哪）、**相对短板**（比别人弱/有顾虑在哪）、**匹配分（1-5）**。强项短板都要「相对」—— 是和这一批人比，不是和理想候选人比。
- 给一个**结论**：谁更合适，以及「如果客户更看重 X 就选 A、更看重 Y 就选 B」这种带条件的建议，比单选一个更有用。

## 输出格式

```
**对比结论**：<一句话 + 带条件的建议>

| 候选人 | 相对强项 | 相对短板 | 匹配分 |
|--------|---------|---------|--------|
| <姓名> | ... | ... | x/5 |
...

**更推荐**：<候选人> —— <理由>
```
