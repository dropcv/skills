# 面试题生成

为某个具体候选人 + 某个具体岗位，设计一份有针对性的面试题，而不是通用题库。

## 何时用

- 要面试某候选人，需要面试官提纲
- 想针对候选人简历里的存疑点 / 亮点准备追问
- 给用人经理准备一份「重点考察什么」的清单

## 前置：接入 DropCV 数据能力

本技能负责出题方法，候选人数据靠 DropCV 数据工具拿。先接入 DropCV MCP 或
Skills API（见 https://docs.dropcv.work/integration/mcp ）。会用到：

- `candidate_profile_read` — 按 candidate_id 读候选人完整档案
- `candidate_search` — 不知道 ID 时按关键词/姓名先搜
- `jd_parse` — 解析目标 JD 文本

## 工作流

1. 读候选人档案（`candidate_profile_read`）。
2. 解析目标 JD（`jd_parse`）。
3. 对照「JD 要求」和「候选人背景」，找出要重点验证的点，按下面四组出题。

## 方法论

好的面试题是「针对这个人」的，目标是验证而不是考倒人。分四组：

- **经历验证**：候选人简历里写的关键成果，挖深 —— 让他讲清楚「他具体做了什么、用了什么、结果如何度量」。专挑和 JD 最相关的经历。
- **动机确认**：为什么看机会、为什么是这个岗位、对职业的预期。判断稳定性和匹配度。
- **风险核实**：针对简历里的疑点 —— 频繁跳槽、空窗、职级与年限不符、技能断层等。一个疑点一道题，给候选人解释的机会。
- **场景题**：给一个这个岗位真实会遇到的场景，看他怎么想。考的是岗位胜任力，不是脑筋急转弯。

每道题附一句「考察点」，方便面试官知道为什么问。

## 输出格式

```
**面试重点**：<一句话 —— 这个人最该验证的 2-3 件事>

**经历验证**
- <题> _(考察：...)_
...

**动机确认**
- <题> _(考察：...)_
...

**风险核实**
- <题> _(考察：...)_
...

**场景题**
- <题> _(考察：...)_
...
```
