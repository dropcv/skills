# 候选人推荐语

把一个候选人推荐给客户或用人方时，写一段让对方愿意往下看的推荐语。

## 何时用

- 要把某候选人提交给客户 / HRBP / 用人经理
- 需要一段推荐说明、推荐邮件正文，或推荐要点清单
- 客户问「这个人怎么样、为什么推他」

## 前置：接入 DropCV 数据能力

本技能负责「怎么判断、怎么写」，候选人数据靠 DropCV 数据工具拿。先接入 DropCV
MCP 或 Skills API（见 https://docs.dropcv.work/integration/mcp ）。会用到：

- `candidate_profile_read` — 按 candidate_id 读候选人完整档案
- `candidate_search` — 不知道 ID 时，按关键词/姓名先搜出候选人
- `jd_parse` — 把目标岗位 JD 文本解析成结构化要求（可选）

## 工作流

1. 拿到候选人：有 candidate_id 直接 `candidate_profile_read`；只有名字就先 `candidate_search` 再读档案。
2. 拿到目标岗位：用户给了 JD 文本就 `jd_parse` 成结构化要求；只有岗位名就按经验补全核心要求。
3. 用下面的方法论分析，写出推荐语。

## 方法论

一段好的推荐语回答客户三个问题：**为什么是他、有什么要注意、接下来怎么办。**

- **亮点（3 条）**：挑和这个岗位最相关的 3 个，每条「事实 + 对岗位的意义」。不要罗列简历，要解读。
  - 反例：「在字节做了 5 年后端」
  - 正例：「字节 5 年支撑过亿级流量的交易系统 —— 正好对得上这个岗位要的高并发经验」
- **风险 / 待确认（1-2 条）**：诚实但不劝退。把可能的顾虑摆出来，并给出「怎么验证」。客户最反感的是推过来才发现的坑。
- **下一步建议**：给客户一个明确动作 —— 约面、补充背景调查、或先电话沟通某个点。

语气：克制、专业，像资深猎头跟客户对话，不浮夸、不堆形容词。

## 输出格式

用 Markdown 输出，给人看：

```
**推荐结论**：<一句话 —— 推荐力度 + 核心理由>

**核心亮点**
1. <亮点 + 对岗位的意义>
2. ...
3. ...

**需要注意**
- <风险/待确认点 + 如何验证>

**建议下一步**：<明确动作>
```
